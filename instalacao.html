Guia de Instalação

Instruções passo a passo para instalar e configurar o Sistema de Controle de Manutenção de Colheitadeiras

Requisitos do Sistema

Antes de iniciar a instalação, certifique-se de que seu ambiente atende aos seguintes requisitos:

  • Python 3.8 ou superior
  • pip (gerenciador de pacotes Python)
  • SQLite (para desenvolvimento) ou MySQL (para produção)
  • Navegador web moderno (Chrome, Firefox, Edge ou Safari)

Opções de Instalação

Existem três maneiras de instalar e executar o Sistema de Controle de Manutenção de Colheitadeiras:

1. Instalação Local

Ideal para testes e desenvolvimento. Instala o sistema em seu computador local.

Ver instruções

2. Instalação em Servidor

Para ambientes de produção. Instala o sistema em um servidor Linux com MySQL.

Ver instruções

3. Implantação no Render

Solução hospedada na nuvem. Implanta o sistema na plataforma Render.

Ver instruções

Instalação Local

Siga os passos abaixo para instalar o sistema em seu computador local:

1 Baixe o pacote do sistema

Faça o download do arquivo ZIP contendo o sistema completo na página de download.

2 Extraia os arquivos

Descompacte o arquivo ZIP em uma pasta de sua preferência.

unzip sistema_manutencao_agricola_final.zip -d sistema_manutencao

3 Crie um ambiente virtual Python

É recomendado usar um ambiente virtual para isolar as dependências do projeto:

cd sistema_manutencao
python -m venv venv

# No Windows
venv\Scripts\activate

# No Linux/Mac
source venv/bin/activate

4 Instale as dependências

Instale todas as bibliotecas necessárias usando o pip:

pip install -r requirements.txt

5 Inicialize o banco de dados

Execute o script de inicialização para criar o banco de dados e inserir dados de exemplo:

python init_db.py

6 Inicie a aplicação

Execute o servidor Flask:

python app.py

A aplicação estará disponível em http://localhost:5000

Credenciais de acesso

Use as seguintes credenciais para acessar o sistema:

  • Administrador: admin / admin123
  • Técnico: tecnico / tecnico123
  • Operador: operador / operador123

Instalação em Servidor

Para ambientes de produção, recomendamos a instalação em um servidor Linux com MySQL:

1 Prepare o servidor

Instale os pacotes necessários no servidor:

sudo apt update
sudo apt install python3 python3-pip python3-venv mysql-server nginx

2 Configure o MySQL

Crie um banco de dados e um usuário para o sistema:

sudo mysql -u root -p

CREATE DATABASE manutencao_agricola;
CREATE USER 'manutencao_user'@'localhost' IDENTIFIED BY 'sua_senha';
GRANT ALL PRIVILEGES ON manutencao_agricola.* TO 'manutencao_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;

3 Baixe e configure o sistema

Baixe o sistema e configure-o para usar o MySQL:

cd /var/www
sudo mkdir manutencao_agricola
sudo chown $USER:$USER manutencao_agricola
cd manutencao_agricola

# Baixe e extraia o sistema
wget https://seusite.com/downloads/sistema_manutencao_agricola_final.zip
unzip sistema_manutencao_agricola_final.zip

# Crie e ative o ambiente virtual
python3 -m venv venv
source venv/bin/activate

# Instale as dependências
pip install -r requirements.txt
pip install gunicorn

4 Configure o banco de dados

Edite o arquivo config.py para usar o MySQL:

nano config.py

Substitua a configuração do banco de dados por:

SQLALCHEMY_DATABASE_URI = 'mysql+pymysql://manutencao_user:sua_senha@localhost/manutencao_agricola'

5 Inicialize o banco de dados

python init_db.py

6 Configure o Gunicorn

Crie um arquivo de serviço para o Gunicorn:

sudo nano /etc/systemd/system/manutencao.service

Adicione o seguinte conteúdo:

[Unit]
Description=Gunicorn instance to serve Sistema de Manutencao Agricola
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/var/www/manutencao_agricola
Environment="PATH=/var/www/manutencao_agricola/venv/bin"
ExecStart=/var/www/manutencao_agricola/venv/bin/gunicorn --workers 3 --bind unix:manutencao.sock -m 007 app:app

[Install]
WantedBy=multi-user.target

7 Configure o Nginx

Crie um arquivo de configuração para o Nginx:

sudo nano /etc/nginx/sites-available/manutencao

Adicione o seguinte conteúdo:

server {
    listen 80;
    server_name seu_dominio.com;

    location / {
        include proxy_params;
        proxy_pass http://unix:/var/www/manutencao_agricola/manutencao.sock;
    }

    location /static {
        alias /var/www/manutencao_agricola/static;
    }
}

Ative o site e reinicie o Nginx:

sudo ln -s /etc/nginx/sites-available/manutencao /etc/nginx/sites-enabled
sudo systemctl restart nginx

8 Inicie o serviço

sudo systemctl start manutencao
sudo systemctl enable manutencao
Segurança

Para um ambiente de produção, recomendamos:

  • Configurar HTTPS com Let's Encrypt
  • Alterar as senhas padrão
  • Configurar backups regulares do banco de dados
  • Implementar um firewall

Implantação no Render

O Render é uma plataforma de hospedagem na nuvem que oferece uma maneira simples de implantar aplicações web.

1 Prepare o repositório no GitHub

Crie um novo repositório no GitHub:

  1. Acesse https://github.com/new
  2. Crie um novo repositório (ex: sistema-manutencao-colheitadeiras)
  3. Descompacte o arquivo ZIP do sistema em seu computador
  4. Inicialize um repositório Git local e faça o push para o GitHub:
cd sistema_manutencao
git init
git add .
git commit -m "Versão inicial do Sistema de Manutenção de Colheitadeiras"
git remote add origin https://github.com/SEU_USUARIO/sistema-manutencao-colheitadeiras.git
git push -u origin main
